## 关键代码修改

修改文件：

```bash
/home/orangepi/px4_ws/src/mocap_to_px4_bridge/src/mocap_bridge.cpp
```

本次成功配置中，`mocap_to_px4_bridge` 不再只发布位置，而是通过参数开启姿态四元数发布：

```bash
-p publish_orientation:=true
```

当前桥接逻辑：

- 位置转换保持为 ROS/动捕 FLU 到 PX4 NED：
  - `x` 不变
  - `y` 取反
  - `z` 取反
- NOKOV orientation 原始顺序为 `x, y, z, w`
- PX4 `VehicleOdometry.q` 顺序为 `w, x, y, z`
- 当前姿态最小转换为：
  - `q[0] = qw`
  - `q[1] = qx`
  - `q[2] = -qy`
  - `q[3] = -qz`
- 写入前检查四元数 norm，归一化后发布
- `q[0] < 0` 时整体取反，避免四元数符号跳变
- q 有效时：
  - `orientation_variance = [0.05, 0.05, 0.05]`
- q 无效时：
  - q 和 orientation_variance 仍写 NaN

为了便于回退，`publish_orientation` 默认值保留为 `false`。实际悬停测试时必须显式加 `true`。

## 启动命令

### 1. 启动 Micro XRCE-DDS Agent

```bash
MicroXRCEAgent serial --dev /dev/ttyS6 -b 921600
```

### 2. 启动 NOKOV 节点

```bash
source /opt/ros/foxy/setup.bash
source ~/px4nokov_ws/install/setup.bash

ros2 launch mocap_nokov mocap.launch.py
```

### 3. 启动 mocap 到 PX4 DDS 桥接

本次成功悬停使用的是 `/fmu/in/vehicle_visual_odometry`，不是 `/fmu/in/vehicle_mocap_odometry`。

```bash
source /opt/ros/foxy/setup.bash
source ~/px4_ws/install/setup.bash

ros2 run mocap_to_px4_bridge mocap_bridge --ros-args \
  -p mocap_topic:=/Robot_1/pose \
  -p px4_odom_topic:=/fmu/in/vehicle_visual_odometry \
  -p publish_orientation:=true
```

### 4. 启动 Offboard 控制节点

先拆桨验证 setpoint：

```bash
source /opt/ros/foxy/setup.bash
source ~/px4_ws/install/setup.bash

ros2 run dds_offboard_control dds_offboard_control_node --ros-args \
  -p target_z:=-0.30 \
  -p auto_offboard:=false \
  -p auto_arm:=false
```

拆桨测试 Offboard 切换：

```bash
ros2 run dds_offboard_control dds_offboard_control_node --ros-args \
  -p target_z:=-0.30 \
  -p auto_offboard:=true \
  -p auto_arm:=false
```

确认拆桨流程正常、QGC 状态正常后，再进行带桨低高度悬停测试。

## 验证命令

ROS 2 Foxy 不支持 `ros2 topic echo --once`，所以使用 `timeout`。

### 检查外部视觉输入

```bash
timeout 3s ros2 topic echo /fmu/in/vehicle_visual_odometry
```

期望看到：

```text
position: 有效数值
q: 非 NaN
orientation_variance:
- 0.05
- 0.05
- 0.05
```

本次实测示例：

```text
q:
- 0.9992
- -0.0072
- 0.0005
- 0.0389
orientation_variance:
- 0.05
- 0.05
- 0.05
```

### 检查 PX4 本地位置状态

```bash
timeout 5s ros2 topic echo /fmu/out/vehicle_local_position | grep -E "xy_valid|z_valid|v_xy_valid|v_z_valid|dead_reckoning|heading|heading_good_for_control"
```

成功状态：

```text
xy_valid: true
z_valid: true
v_xy_valid: true
v_z_valid: true
heading_good_for_control: true
dead_reckoning: false
```

本次实测中 heading 稳定在约：

```text
heading: 0.078 rad
heading_good_for_control: true
dead_reckoning: false
```

### 检查 Offboard setpoint

```bash
timeout 3s ros2 topic echo /fmu/in/trajectory_setpoint
timeout 3s ros2 topic echo /fmu/in/offboard_control_mode
```

当前 Offboard 节点逻辑：

- `x/y` 固定为进入节点时的当前位置
- `z = -0.30`
- `yaw = NaN`
- 使用 position setpoint

## 关键排查过程

1. 最初只发布 position，q 全为 NaN。
2. PX4 中 `heading_good_for_control` 一直为 false。
3. 尝试直接加入 q 后，曾出现 EKF 水平位置融合异常。
4. 回退 q 后，位置融合恢复，但 heading 仍不能用于控制。
5. 确认 EKF2 实际融合的是 `/fmu/in/vehicle_visual_odometry`，而不是 `/fmu/in/vehicle_mocap_odometry`。
6. 将桥接发布目标改为 `/fmu/in/vehicle_visual_odometry` 后，位置融合恢复。
7. 给桥接增加 `publish_orientation` 参数，默认关闭，测试时显式开启。
8. 开启 q 发布后，PX4 输入 q 和 orientation_variance 有效。
9. 调整 PX4 外部视觉 yaw 融合后，`heading_good_for_control=true`。
10. 拆桨 Offboard 测试通过。
11. 带桨低高度悬停测试成功。

## 当前推荐复现实验流程

每次飞行前按以下顺序确认：

1. 拆桨或确保安全状态
2. 启动 Micro XRCE-DDS Agent
3. 启动 NOKOV 节点
4. 启动 mocap 桥接，并确认带 `publish_orientation:=true`
5. 检查 `/fmu/in/vehicle_visual_odometry` 中 q 非 NaN
6. 检查 `/fmu/out/vehicle_local_position`
7. 只有当以下条件全部满足时，才允许进入 Offboard 测试：

```text
xy_valid: true
z_valid: true
v_xy_valid: true
v_z_valid: true
heading_good_for_control: true
dead_reckoning: false
```

## 安全记录

- 在 `heading_good_for_control=false` 时，不进行带桨 Offboard 悬停。
- 在 `dead_reckoning=true` 时，不进行带桨 Offboard 悬停。
- 在 `xy_valid=false` 或 `z_valid=false` 时，不进行带桨 Offboard 悬停。
- 每次重启飞控或重新启动桥接后，都需要重新验证 q、position 和 local position 状态。
